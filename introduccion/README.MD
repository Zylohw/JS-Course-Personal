# DUDAS A RESOLVER

## ¿ Que es JavaScript?
- Es un lenguaje de desarrollo web, este lenguaje permite trabajar en varios ambitos de la programación 
- Progrador: Front-End, Back-End, FullStack , Mobile e inlcuso desarrollador de apps de escritorio, tambien en microcontroladores
- Dicen que JS es una navaja suiza de los programadores debido a la gran cantidad de caminos que uno puede seguir en el desarrollo. Incluso de la mayoria de aplicaciones que usamos hoy en dia estan contruidas usando liberias o frameworks que usan Javascript un ejemplo común es Intragra, Facebook y WhatsApp que son desarrollados con React una libreria que es propiedad e de Meta. Mas tambien podemos usar otros frameworks mas pesados como: Vue, Express, Angular, Ionic,React.

## Framework vs Liberia
- Framework: Entrega herramientas y son varias para que uno pueda desarrollador apps complejas este mismo te dira como trabajar 
- Libreria: se encarga de solucionar un problema en especifico 
- entonces el framework entregra varias herramientas que uno puede usar mas la libreria se enfocara en solucionar uno solo pero tambien es considerado una herramienta

## ¿Que es ECMAScript? 
- Cuando se desarrollo JS se hizo con el fin de navegar en sitios dinamicos. Este se hizo un directamente para ejecutar en exploradores web como Chrome, Brave , Edge y Firefox. 
- Cuando Nestcape (empresa que desarrolló JS) decido entregar JS a la organzación ECMA que anteriormente era conocido bajo el acronimo (European Computer Manufactures Associations) hoy en dia ya no es un acronimo si no que ahora se llama ECMA que ahora es una organización si fines de lucros y ahora contribuyen con estandares tecnologicos unos ejemplos son:
      * ECMA 334 → C#
      * ECMA 404 → Especificaciones JSON
      * ECMA 408 → Dart
      * ECMA 2015 → Introducción de funcinalidades como  `let` y `const`, se invento la palabra `class`
> **NOTA**: Solo es historia pero es necesaria.
- Bien, actualmente los expladores web tiene un motor encargado de ejecutar JS, el motor de Google Chomre se llama V8 y es de codigo de abierto (cualquiera puede ver el codigo fuente de este mismo) y gracias a eso permito que Ryan Dahl pudiera embeber el codigo de V8 en C++ y gracias a esto permitio que el codigo de JS se pueda ejecutar y esto produjo que JS se pueda ejecutar en varios entorno como: Web, Mobile, Servidores, apps de escritorio, micro-controladores y este codigo de C++ se le llama NodeJS.
